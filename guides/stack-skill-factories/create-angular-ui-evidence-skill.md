# Create Angular / UI Evidence Skill

## Stack Scope
Use when the target project has a frontend and needs visual, journey, screenshot, or browser evidence.

## Required Project Discovery
UI framework, app start commands, test commands, screenshot tooling, viewport matrix, accessibility expectations, evidence paths.

## Skill Drafting Steps
1. Discover how the target UI runs.
2. Define viewports and user journeys.
3. Define screenshot/video evidence rules.
4. Redact sensitive visual content.

## Command Safety
Use only target start/test commands. Do not assume Angular unless the target project actually uses it.

## Verification
Run target UI smoke checks, screenshot checks, or document blockers.

## Anti-Patterns
Treating one screenshot as full UX proof; publishing sensitive screenshots.
