# Enterprise Completion Matrix Template

## Intended Architecture
Use for projects that need multi-workstream readiness, launch tracking, or enterprise delivery governance.

## Project Inputs
Workstreams, acceptance criteria, evidence types, owners, approvers, blockers.

## Adaptation Questions
- What means complete?
- Who reviews each stream?
- Which evidence supports but does not approve completion?
- What blockers prevent launch?

## Template Rules
Define completion rows, evidence expectations, reviewer ownership, blocker states, and re-review loops.

## Outputs
Completion matrix, acceptance checklist, blocker ledger, review plan.

## Review Gates
Evidence artifacts are not treated as approval; project-specific master plans are not copied.

## Non-Goals
Does not import one project's enterprise plan as a universal policy.
