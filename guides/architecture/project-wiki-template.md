# Project Wiki Template

## Intended Architecture
Use for projects with hand-authored docs, generated docs, knowledge graphs, or documentation maps.

## Project Inputs
Canonical docs, generated folders, code symbols, owners, freshness markers, relation types.

## Adaptation Questions
- Which docs are source of truth?
- Which docs are generated?
- How are stale docs detected?
- Which graph relations matter?

## Template Rules
Define documentation hierarchy, freshness checks, generated-doc policy, and knowledge graph sections.

## Outputs
Wiki structure, source-of-truth map, graph relation rules, generated-doc classification.

## Review Gates
Generated wiki content is not promoted to authority without review.

## Non-Goals
Does not copy generated wiki-domain exports or local generated docs into the reusable repository.
