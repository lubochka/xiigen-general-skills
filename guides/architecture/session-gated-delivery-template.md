# Session Gated Delivery Template

## Intended Architecture
Use for structured delivery sessions with explicit gates, evidence, and handoffs.

## Project Inputs
Session list, goals, allowed commands, review owner, evidence policy, output contract.

## Adaptation Questions
- What opens broad build/test/deploy gates?
- What must be reviewed before stop?
- What is the handoff format?
- Which session content is architecture, stack, or legal/content?

## Template Rules
Define session scope, gate transitions, evidence requirements, and handoff checklist.

## Outputs
Session protocol, gated command policy reference, handoff checklist, review records.

## Review Gates
Source `SESSION-00..14` files are parameterized, not copied as-is.

## Non-Goals
Does not publish private session notes, evidence logs, or stack commands.
