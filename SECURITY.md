# Security Policy

Please do not report security vulnerabilities in public issues, pull requests,
comments, discussions, screenshots, logs, or social media.

## Reporting

Report suspected vulnerabilities through a private channel controlled by the
project maintainers. Before public release, maintainers must publish a
monitored security contact or enable GitHub private vulnerability reporting.

If no private reporting channel is available, do not include exploit details,
secrets, source code, customer data, regulated data, or confidential project
materials in a public report. Open a minimal public issue asking maintainers to
provide a private security contact.

## Scope

Security reports may cover this repository's reusable skills, templates,
adaptation guides, review checklists, scripts, examples, and GitHub community
templates.

## Sensitive Material

Do not include passwords, API keys, private keys, tokens, customer source code,
private prompts, regulated data, logs containing secrets, screenshots with
confidential material, or internal evidence bundles in public issues, pull
requests, screenshots, logs, or discussions.

## Responsible Disclosure

Please give maintainers a reasonable opportunity to investigate and remediate
before public disclosure. This project does not currently operate a paid bug
bounty program.
